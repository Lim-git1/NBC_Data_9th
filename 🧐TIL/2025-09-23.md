```sql
-- [1. 서브쿼리 버전 코드]

SELECT
  m.member_name,
  r.review_text,
  DATE_FORMAT(r.review_date, '%Y-%m-%d') AS review_date
FROM REST_REVIEW r
JOIN (
  -- 최다 리뷰 회원만 뽑아놓은 파생 테이블
  SELECT rc.member_id
  FROM (
    -- 회원별 리뷰 수
    SELECT member_id, COUNT(*) AS cnt
    FROM REST_REVIEW
    GROUP BY member_id
  ) rc
  JOIN (
    -- 최댓값
    SELECT MAX(cnt) AS mx
    FROM (
      SELECT member_id, COUNT(*) AS cnt
      FROM REST_REVIEW
      GROUP BY member_id
    ) x
  ) mm
  ON rc.cnt = mm.mx
) t  USING (member_id)
JOIN MEMBER_PROFILE m USING (member_id)
ORDER BY r.review_date ASC, r.review_text ASC;

```

```sql
-- [2. CTE 버전 코드]

-- 1) 회원별 리뷰수
WITH review_counts AS (
  SELECT member_id, COUNT(*) AS cnt
  FROM REST_REVIEW
  GROUP BY member_id
),
-- 2) 최다 리뷰 회원 id
top_members AS (
  SELECT member_id
  FROM review_counts
  WHERE cnt = (SELECT MAX(cnt) FROM review_counts)
)

SELECT
  m.member_name,
  r.review_text,
  DATE_FORMAT(r.review_date, '%Y-%m-%d') AS review_date
FROM REST_REVIEW r 
JOIN top_members t USING (member_id)
JOIN MEMBER_PROFILE m USING (member_id)
ORDER BY r.review_date ASC, r.review_text ASC;

```

```sql
-- [3. 윈도우함수(RANK) 버전 코드]

-- 회원별 리뷰 수 집계 후 순위 매기기
WITH review_counts AS (
  SELECT member_id, COUNT(*) AS cnt
  FROM REST_REVIEW
  GROUP BY member_id
),
ranked AS (
  SELECT member_id, cnt,
         RANK() OVER (ORDER BY cnt DESC) AS rnk
  FROM review_counts
)
SELECT
  m.member_name,
  r.review_text,
  DATE_FORMAT(r.review_date, '%Y-%m-%d') AS review_date
FROM ranked k
JOIN REST_REVIEW r USING (member_id)
JOIN MEMBER_PROFILE m USING (member_id)
WHERE k.rnk = 1
ORDER BY r.review_date ASC, r.review_text ASC;

```

### 비교 Point
1. **서브쿼리 버전 코드**
   - 가독성↓: 괄호 속 괄호… 중첩이 깊어지면 읽기/유지보수 난이도 상승.
2. **CTE 버전 코드**
   - 모듈화 → 가독성, 디버깅, 재사용, 유지보수 용이
3. **윈도우함수(RANK) 버전 코드**
   - 읽기 쉬운 표준 패턴이라 팀 협업에 유리.

### 코드 패턴 Point

- [비효율적] **조인 → 집계**
  - 모든 행 기준으로 조인된 뒤에야 집계가 일어나기 떄문에 불필요한 메모리 사용 유발
- [효율적] **집계(CTE) → 축소(필터링) → 조인**
  - 데이터를 축소시키고 조인하기 때문에 메모리 비용 절감

### [아티클 스터디] : [A/B 테스트 제대로 이해하기 : ② A/B 테스트를 위한 기초 통계 이해하기](https://yozm.wishket.com/magazine/detail/1644/)

- **요약** :
    - 모집단 : 전체 / 표본 : 일부
        - ‘일부’를 통해 ‘전체’를 추측 → ‘우연’발생
    - 귀무가설 : 평소 상태 / 대립가설 : 평소와 반대되는 상태
    - 단측검정 : 특정 방안의 우위 또는 열세를 가정
    - 양측검정 : 우위는 모르겠고 일단 차이가 있다는 가정
    - 신뢰수준 : 추측이 어느 정도로 신뢰할 수 있는가?
    - 신뢰구간 : 실험 결과가 실제로 포함될 것으로 예측하는 범위
    - 점추정 / 구간추정
    - 표준오차 : 표본 집단마다 생기는 차이
    - P-value : 실험 결과가 우연인지 아닌지 확인

- **주요 포인트** :
    - 가설을 통한 실험은 우연히 발생할 수밖에 없기 때문에 적절한 기초 통계를 활용해 원하는 답을 찾아야한다.

- **인사이트** :
  - 글에서 귀무가설과 대립가설 설명을 읽으면서, 세션 때는 ‘대립가설은 내 주장, 귀무가설은 내 주장에 반대되는 주장’ 정도로만 외웠는데, 실제 A/B 테스트에 적용되는 예시를 보니 왜 이 구분이 중요한지 좀 감을 잡을 수 있었다.
