### [SQL] : 어쩌구 함수의 사용법
```sql
-- [JOIN을 사용한 코드 버전] --> 틀림
SELECT date_format(SALES_DATE, '%Y-%m-%d')
        , PRODUCT_ID
        , CASE WHEN offline.sales_amount is null
        THEN online.user_id
        WHEN offline.sales_amount is not null
        THEN "NULL"
        END 'USER_ID'
        , CASE WHEN offline.sales_amount is null
        THEN online.sales_amount
        WHEN offline.sales_amount is not null
        THEN offline.sales_amount
        END 'SALES_AMOUNT'
FROM online_sale online left join offline_sale offline
USING (sales_date, product_id)
WHERE sales_date like '2022-03%'
ORDER BY sales_date asc, product_id asc, online.user_id asc
```
- 내 시도 : 
  - 한동안 JOIN 문제를 계속  풀었더니, 문제를 딱 보고 테이블이 두 개 이상이면 냅다 `JOIN`을 때려버리는 습관이 생겼다.
- 문제 : 
  - `online` 테이블에는 22년 3월 data가 있었으나, `offline` 테이블에는 22년 3월 data가 없다. 이 상황에서 `JOIN`을 하니 컬럼을 동일하게 가져갈 수 없는 상황이 되었다.
  

```sql
-- [UNION을 사용한 코드 버전] --> 맞음
(SELECT date_format(SALES_DATE, '%Y-%m-%d') SALES_DATE
        , PRODUCT_ID
        , USER_ID
        , SALES_AMOUNT
FROM online_sale
WHERE sales_date like '2022-03%')
UNION ALL
(SELECT date_format(SALES_DATE, '%Y-%m-%d') SALES_DATE
        , PRODUCT_ID
        , NULL 'USER_ID'
        , SALES_AMOUNT
FROM offline_sale
WHERE sales_date like '2022-03%')
ORDER BY sales_date asc, product_id asc, user_id asc
```


### → 다시 되세이는 `JOIN` VS `UNION`

- `JOIN`이 필요한 경우 : **열**을 기준으로 데이터를 좌우로 합칠 때
  - 두 테이블에 **공통된 컬럼**이 있을 때 사용
  
- `UNION`이 필요힌 경우 : **행**을 기준으로 데이터를 위아래로 합칠 때
  - 두 쿼리의 컬럼 수와 타입이 동일해야 함

### [이론] : $\bar{X}$ vs $\bar{x}$

- $\bar{X}$ : 확률변수로서의 표본평균
  - 아직 표본을 뽑기 전, 표본평균이 가질 수 있는 분포를 말할 때 사용
  
- $\bar{x}$ : 실제로 관측된 값으로서의 표본평균
  - 이미 표본을 뽑아 계산한 결과값
